Graph Databases — Overview

article source: wikipedia.org
A Graph Database (obviously) uses graph structures to allow for semantic queries. Graph databases are a form of NoSQL databasing that was created to be useful beyond the limitations of relational approaches. However, Graph Databases can be a natural fit for storing application data for a wide variety of software. A Graph has nodes, edges and properties to represent the information stored within it.
The data items that a graph database works with are collections of nodes and edges. The edges link properties or relationships between nodes. The nodes represent entities within the database. The graph model explicitly arranges the dependencies between data-nodes. Relational databases and other NoSQL approaches link data using implicit connections. By design, graph databases allow simple and quick querying of complex data structures. Graph databases can be visualized clearly because of their network-like arrangement.
